# Verification: S2-03 Tenant selection wall

## Objective
Implement UI and Logic for selecting a clinic when user has multiple memberships.

## 1. Routes
**Command:** `grep -A 5 "Clinic Selection Wall" app/Config/Routes.php`
**Result:**
```php
$routes->group('clinic', ['filter' => 'auth'], function($routes) {
    $routes->get('select', 'ClinicSelector::select');
    $routes->post('select', 'ClinicSelector::processSelect');
    $routes->get('no-clinic', 'ClinicSelector::noClinic');
    $routes->post('switch', 'ClinicSelector::switchClinic');
});
```

## 2. Controller Logic (`ClinicSelector`)
- `select()`: Checks membership count again. If 1, auto-redirects. If >1, shows view.
- `processSelect()`: Validates `clinic_id` against `clinic_users` table for current user. Sets session keys. Regenerates ID. Redirects to dashboard.

## 3. Views
- `app/Views/clinic/select.php`: Lists available clinics as buttons. POSTs to `processSelect`.
- `app/Views/clinic/no_clinic.php`: Message and Logout link.
