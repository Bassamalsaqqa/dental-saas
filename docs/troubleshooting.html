Troubleshooting Guide

Common issues and solutions for DentaCare Pro

Quick Diagnostics

Before diving into specific issues, run these quick diagnostics to identify common problems:

System Check

  • PHP version ≥ 7.4
  • MySQL service running
  • Web server running
  • File permissions correct

Application Check

  • Database connection
  • .env file configured
  • Writable directories
  • URL rewriting enabled

Installation Issues

Database Connection Error

Error: "Unable to connect to your database server using the provided settings"

Possible Causes:

  • • Incorrect database credentials in .env file
  • • MySQL service not running
  • • Database doesn't exist
  • • User doesn't have proper permissions

Solutions:

1. Check .env file:

database.default.hostname = localhost database.default.database = dentacare_pro database.default.username = your_username database.default.password = your_password

2. Verify MySQL service:

# Check if MySQL is running sudo systemctl status mysql # Start MySQL if not running sudo systemctl start mysql

3. Test database connection:

mysql -u your_username -p -h localhost dentacare_pro

Permission Denied Errors

Error: "Permission denied" or "Unable to write to directory"

Solutions:

1. Set correct permissions:

# Set permissions for writable directories chmod -R 755 writable/ chmod -R 755 public/assets/ # Set ownership (replace www-data with your web server user) chown -R www-data:www-data writable/ chown -R www-data:www-data public/assets/

2. Check .env file permissions:

chmod 644 .env chown www-data:www-data .env

404 Not Found Errors

All pages return "404 Not Found" except the homepage

Solutions:

1. Check .htaccess file (Apache):

# Ensure .htaccess exists in public directory RewriteEngine On RewriteCond %{REQUEST_FILENAME} !-f RewriteCond %{REQUEST_FILENAME} !-d RewriteRule ^(.*)$ index.php/$1 [L]

2. Enable mod_rewrite (Apache):

# Enable mod_rewrite sudo a2enmod rewrite sudo systemctl restart apache2

3. Check Nginx configuration:

location / { try_files $uri $uri/ /index.php?$query_string; }

Security Verification

Ensure your installation is secure by verifying the following:

1. Web Server Rules

Accessing /.env or /app/Config/Database.php via browser must return 403 Forbidden.

2. Secrets Management

Ensure .env is NOT in the webroot and database credentials are NOT hardcoded in files.

Runtime Issues

Login Issues

Cannot log in with generated credentials.

Solutions:

1. Reset admin password (CLI/SQL only):

# Run this SQL query in your database UPDATE users SET password = 'YOUR_GENERATED_BCRYPT_HASH' WHERE email = 'your_admin_email@example.com'; # Note: You must generate a valid bcrypt hash for your new password.

2. Check session configuration:

# In .env file app.sessionDriver = 'CodeIgniter\Session\Handlers\DatabaseHandler' app.sessionSavePath = 'ci_sessions'

3. Clear browser cache and cookies

Performance Issues

Slow page loading or database queries taking too long

Solutions:

1. Enable query caching:

# In .env file database.default.cacheOn = true database.default.cacheDir = 'writable/cache/database/'

2. Add database indexes:

# Add indexes for frequently queried fields ALTER TABLE patients ADD INDEX idx_email (email); ALTER TABLE appointments ADD INDEX idx_date (appointment_date); ALTER TABLE examinations ADD INDEX idx_patient_date (patient_id, examination_date);

3. Clear application cache:

# Clear cache files rm -rf writable/cache/*

Email Issues

Appointment reminders or notifications not being sent

Solutions:

1. Configure SMTP settings:

# In .env file email.SMTPHost = 'smtp.gmail.com' email.SMTPUser = 'your-email@gmail.com' email.SMTPPass = 'your-app-password' email.SMTPPort = 587 email.SMTPCrypto = 'tls'

2. Test email configuration:

# Create a test email script $email = \Config\Services::email(); $email->setTo('test@example.com'); $email->setSubject('Test Email'); $email->setMessage('This is a test email'); $email->send();

Data Issues

Missing Data

Patient records, appointments, or other data not displaying correctly

Solutions:

1. Check database import:

# Verify tables exist SHOW TABLES; # Check table structure DESCRIBE patients; DESCRIBE appointments;

2. Re-import sample data:

# Drop and recreate database DROP DATABASE dentacare_pro; CREATE DATABASE dentacare_pro; USE dentacare_pro; SOURCE democa_dental.sql;

Data Corruption

Data appears corrupted or displays incorrectly

Solutions:

1. Check character encoding:

# Ensure database uses UTF-8 ALTER DATABASE dentacare_pro CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

2. Repair database tables:

# Repair corrupted tables REPAIR TABLE patients; REPAIR TABLE appointments; REPAIR TABLE examinations;

Browser Issues

JavaScript Errors

Interactive features not working or console showing errors

Solutions:

1. Check browser console:

Press F12 and check the Console tab for JavaScript errors

2. Clear browser cache:

Hard refresh the page (Ctrl+F5 or Cmd+Shift+R)

3. Check file permissions:

# Ensure JS files are readable chmod 644 public/assets/js/*

CSS Issues

Page styling appears broken or missing

Solutions:

1. Check CSS file loading:

Verify CSS files are loading in browser developer tools

2. Rebuild Tailwind CSS:

# Rebuild Tailwind CSS npm run build # Or npx tailwindcss -i ./src/input.css -o ./public/assets/css/tailwind.css --watch

Debugging Tools

Enable Debug Mode

Enable debug mode to get detailed error information:

# In .env file CI_ENVIRONMENT = development # Enable error reporting error_reporting = E_ALL display_errors = 1 log_errors = 1

Warning: Only enable debug mode in development environments. Never enable it in production.

Log Files

Check log files for detailed error information:

Application logs:

tail -f writable/logs/log-$(date +%Y-%m-%d).php

Web server logs:

# Apache tail -f /var/log/apache2/error.log # Nginx tail -f /var/log/nginx/error.log

Database Debugging

Enable database query logging:

# In .env file database.default.DBDebug = true database.default.DBLogQueries = true
